Software Life Cycle: development phase The major steps in the traditional software development life cycle are requirments analysis, design, implementation, and testing.
Requirements analysis To specify what services the proposed system will provide, to identify any conditions(time constraints, securtiy, and so on) on those services, and to define how the outside world will interact with the system
Design Creates a plan for the construction of the proposed system
implementation Involves the actual writing of programs, creation of data files, and development of databases
Testing Equated with the process of debugging programs and confirming that the final software product was compatible with the software requirments specifications
modular implementation a module can consist of a single file that combines the module interface and implementation. You can also put the implementations in one or more separate module implementation files, similar to how.hand.cppfiles are used.
Unified Modeling Language (UML) is a general purpose modelling Language, to define a standard way to visualize the way a system has been designed.
use case a specific situation in which a product or service could potentially be used
use case diagram Illustrates a set of use cases for a system
data relationship A single entity that contains multiple fields
one-to-one the link between the information in two table
one-to-many the relationship between two entities
many-to-many a table relationship
design pattern general solution to the commonly occurring problems in software designing.
Software quality assurance (SQA) is a means and practice of monitoring the software Engineering processes and methods used in a project to ensure proper quality of the software.
glass box testing esting technique that examines the program structure and derives test data from the program logic/code
black box testing Black Box Testing is a software testing method in which the functionalities of software applications are tested without having knowledge of internal code structure, implementation details and internal paths. Black Box Testing mainly focuses on input and output of software applications and it is entirely based on software requirements and specifications.
beta testing user acceptance testing where the product team gives a nearly finished product to a group of target users to evaluate product performance in the real world.
alpha testing performed using both black and white box testing techniques. It is the first round of testing a new product or software solution goes through, it is concerned with finding bugs, issues or mistakes prior to launch.
