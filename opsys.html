Operating Systems Terminology

Term Definition
operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment
job queue a list of tasks to be carried out in turn by an operating system
FIFO objects in queue are removed in the order in which they arrived
interactive processing allows the user to directly interact with the operating system whilst one or more programs are running.
time_sharing an operating system that provides service to multiple users at the same time
multiprogramming time is divided into intervals and then the execution of each job is restricted to only one interval at a time
applications software consists of the programs for performing tasks particular to the machine's utilizations
system software performs tasks that are common to computer systems in general
graphical user interface (GUI) in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons
shell Older interfaces that communicated with users through textual messages using a keyboard and monitor screen
kernel internal part if an operating system
device driver the software units that communicate with the controllers (or at times, directly with peripheral devices) to carry our operations on the peripheral devices attached to the machine.
file manager job is to coordinate the use of the machine's mass storage facilities
memory manager is charged with the task of coordinating the machine's use of main memory
virtual memory large fictional memory space created by paging
boot strapping is performed by a computer each time it is turned on
ROM special nonvolatile memory cells
deadlock the condition in which two or more processes
spooling holding data for output at a later but more convenient time
super user (or administrator accounts that are established by a single person
multi-factor authentication adding another step to verify a user with a weak password
